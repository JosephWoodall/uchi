# Neuro-Symbolic Inspector

When the user asks to debug the engine, inspect a query, or see the internal "thoughts" or components between the query and response, use this skill.

## 1. Execute the Inspector
Run the `scripts/neuro_inspector.py` script. 

To run a predefined suite of conversational and coding benchmarks:
```bash
python scripts/neuro_inspector.py
```

To debug a specific query, pass the exact query wrapped in quotes:
```bash
python scripts/neuro_inspector.py "What is the capital of France?"
```

## 2. Analyze the Output
The inspector will output the exact sequential steps the engine takes:
1. **Tokens**: How the OmniTokenizer split the query.
2. **Intent Vector**: The dimensionality of the extracted vector from the SSM.
3. **Tool Routing**: The Cosine Similarities against all registered skills.
4. **MCTS Hallucination**: A sample of 5 raw, blind candidates generated by the Trie.
5. **Oracle Execution**: Whether each of the 5 candidates passed the binary coherence filter.
6. **Geometric Selection**: The final chosen candidate based on geometric closeness to the query vector.

Read this output and summarize to the user EXACTLY where the failure or success occurred. Did the Oracle murder all candidates? Was the Trie hallucinating garbage? Did it route to the wrong tool? Explain the engine's internal state to the user.
